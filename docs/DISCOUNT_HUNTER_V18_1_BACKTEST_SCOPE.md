# DCA 折價獵人 V18.1 回測範圍

**Status:** Research / Backtest Only  
**Updated:** 2026-07-07  
**Owner:** Josh

---

## 1. 目的

V18.1 的目的不是立刻改買點，也不是新增正式資金標的。

本階段只做：

```text
用歷史價格驗證折價分層是否合理
找出哪些標的適合深折扣策略
排除只靠短線題材、資料不足或風險過高的標的
```

---

## 2. 正式回測主體

核心回測仍以折價獵人既有框架為主：

```text
BTC
NVDA
TSM
AMD
AVGO
MRVL
VRT
RKLB
LITE
SPCX
```

規則：

```text
BTC：Cycle High 回撤
xStocks / 股票：52週高點回撤
SPCX：上市以來高點回撤
```

---

## 3. 新增觀察標的

使用者指定新增兩個加密高波動觀察標的：

```text
M / MUSDT：MemeCore
BP / BPUSDT：Backpack
```

定位：

```text
只進入回測觀察
不進正式持倉
不進 Telegram 買點推播
不給資金建議
不與 BTC / xStocks 共用正式買點結論
```

---

## 4. 為什麼不能直接放進正式折價獵人？

Meme / 新上市加密幣有三個問題：

```text
上市時間短
歷史高低點樣本不足
可能存在籌碼集中與流動性風險
```

因此它們必須獨立成：

```text
加密高波動觀察組
```

而不是直接併入 BTC 或 xStocks。

---

## 5. 回測資料來源規則

優先順序：

```text
1. Binance 可取得的日線 K 線
2. Binance Alpha / Web3 可匯出的歷史價格
3. 使用者手動匯出的 CSV
```

注意：如果 Binance 公開 Spot API 無法取得 K 線，則不得假裝已完成完整回測。

這類標的只能標記為：

```text
DATA_PENDING
```

---

## 6. 加密高波動觀察組初步買點假設

此組不使用 BTC 的 Cycle High 規則，也不使用股票的 52 週高點規則。

暫定使用：

```text
上市以來高點回撤
```

初步分層僅供回測，不可視為正式買點：

```text
第一層：-50%
第二層：-65%
第三層：-80%
第四層：-90%
```

原因：

```text
Meme / 高波動幣 -30% 至 -50% 可能只是正常波動
過早分批容易被連續下跌吞掉
```

---

## 7. 通過條件

M / BP 若要從觀察組升級，至少需要：

```text
可取得足夠日線資料
至少經歷一次完整上漲與回撤循環
回測後 30 / 60 / 90 日有明確正期望
最大不利跌幅可接受
流動性與持幣集中風險可接受
```

未通過前：

```text
不加入正式折價獵人買點
不進每日 Telegram
不進自動紙上交易
不進資金配置
```

# Josh Portfolio — DCA 與共用逢低池策略 V1

- 定案日期：2026-07-15
- 預定開始執行：2026-08
- 適用範圍：QQQ、NVDA、TSM、AVGO、GOOGL、BTC
- 狀態：已定案，待 2026-08 開始執行
- 最高原則：DCA 是主引擎；逢低買進是額外加速器。兩者資金不得互相挪用。

## 一、策略定位

1. 基礎 DCA 按月持續執行，不因價格下跌、跌破均線或預期更低價格而停扣、減扣或延後。
2. 價格訊號只用於決定是否額外加碼，不用於決定是否停止基礎 DCA。
3. 只有 Quality Gate／投資邏輯失效，才能影響個股的 DCA 資格。
4. 逢低買進使用獨立資金池，不從原定 DCA 預算中挪用。
5. DCA 與逢低買進必須分開記帳、分開統計成本與績效。

## 二、基礎 DCA

### 1. 適用六檔

- QQQ
- NVDA
- TSM
- AVGO
- GOOGL
- BTC

### 2. 每月總額

- 每月基礎 DCA 總預算：100U。
- 100U 為整體月度預算，不代表六檔平均分配，也不代表目前已正式定案每檔金額。
- 各檔正式權重需另外完成重疊曝險檢查後再定案。

### 3. QQQ 重疊原則

QQQ 不是傳統防禦資產，而是較個股分散、但仍高度集中科技成長因子的指數引擎。

正式分配時必須考慮 QQQ 內含的 NVDA、GOOGL、AVGO 等曝險，避免把 QQQ 與個股視為完全獨立的六個曝險。

## 三、共用逢低池

### 1. 池型

- 六檔共用一個逢低池，不設每檔獨立袖套。
- 目的：提高資金利用率，避免資金長期鎖死在尚未觸發的單一資產。

### 2. 建池與維持

- 建池目標：150U～200U。
- 目標上限：200U。
- 維持期每月補水：10U。
- 當池內達 200U 時，當月新增 10U 不再進入逢低池，轉入基礎 DCA。
- 當池內低於 200U 時，恢復每月補水 10U。

### 3. 閒置資金停泊

逢低池平時可放在高流動性、可快速贖回的穩定幣活期收益工具中。

優先順序：

1. 隨時可用
2. 本金穩定
3. 平台與資產分散
4. 收益率

不使用雙幣理財、長期鎖倉、高槓桿套利、流動性挖礦或需等待多日贖回的產品作為主要逢低池。

## 四、單次逢低買入規則

- 基本單位：5U。
- 階梯：1／2／4／8 倍。

| 層級 | 金額 |
|---|---:|
| D1 | 5U |
| D2 | 10U |
| D3 | 20U |
| D4 | 40U |

- 單一資產完成一輪完整階梯合計：75U。
- 現有各資產 D 層門檻與高點模型不因本次定案改變。
- BTC 仍使用正式週期高點模型，不改用股票 52 週高點模型。

## 五、多檔同時觸發的機械排序

同一檢查週期最多執行兩筆新層級，其他進入延遲佇列。

排序順序：

1. 延遲中的訊號優先於新觸發訊號。
2. D 層較深者優先：D4 > D3 > D2 > D1。
3. 同層時，超過該層門檻較多者優先。
4. 再比較 Quality Gate 分數；僅能使用當時可得的分數，禁止使用今天的分數回填歷史。
5. 若仍相同，較早觸發者優先。
6. 最後使用固定代碼順序作為機械化平手規則。

### 延遲訊號處理

- 不取消延遲訊號。
- 下次檢查仍符合該層時，優先於新訊號。
- 若已進入更深層，升級為更深層待執行。
- 若價格回到門檻以上，該延遲訊號失效。

## 六、推播與執行邏輯

- 接近下一層時維持現行推播邏輯。
- 接近訊號顯示距離、進度與觸發後金額，但不代表已正式買入。
- 正式達到 D 層時，才列為可執行訊號。
- 所有訊號仍需 Josh 確認；系統維持 DRY RUN，不自動下單。
- 現行 D 層規則、推播門檻與正式十檔清單不得因本次文件自行變更。

## 八、回測依據

V8 六檔共用逢低池測試：

- 六檔：QQQ、NVDA、TSM、AVGO、GOOGL、BTC
- 基礎 DCA：每檔 10U／月，僅為測試假設
- 共用池階梯：5／10／20／40U
- 每日最多執行兩筆
- 現金收益代理：^IRX
- BTC：running ATH proxy，尚非正式週期高點資料

主要結果：

- 10U／月補水＋200U上限：滿足率約 97.1%。
- 20U／月＋150U上限：滿足率同樣約 97.1%，但投入池中的累計資金更多。
- 75U上限對六檔同步下跌明顯不足。
- 150U開始具實用性；200U最能以低補水速度保存跨週期火力。
- 歷史成功執行 66 次，逢低實際投入約 645U。
- 未執行的兩次為訊號在高點重置前失效，不單純是資金不足。

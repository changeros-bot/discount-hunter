## 1. Principle

槓桿獵人第一階段只使用 Binance 股票代幣 / xStocks / bStocks 名單內已有標的。

不納入：

- 台股 ETF：00631L / 00647L 等先移除。
- 傳統美股槓桿 ETF：SSO / QLD / TQQQ / SOXL 等先移除，除非 Binance 已上架對應代幣且另行驗證。
- 非代幣化個股。
- 反向 ETF。

---

## 2. Core Rule

槓桿 / 波段只找：

```text
健康
股價高
股性活潑
有成交量
代幣流動性可接受
```

槓桿獵人不是折價獵人的加速版，不是找便宜爛股反彈。

---

## 3. Selection Filters

| 條件 | 中文定義 | 初期判斷 |
|---|---|---|
| 健康 | 公司基本面不能明顯破壞 | Quality 至少不是失敗 |
| 股價高 | 避免低價垃圾股 | 原股價格具備機構參與度 |
| 股性活潑 | 適合波段 | 歷史波動 / 回撤 / 反彈幅度足夠 |
| 有量 | 原股與代幣都要有量 | Binance 代幣價格更新與成交正常 |
| 資料正常 | 價格、成本、持倉、API 不可缺失 | 缺資料只能觀察，不能進買點 |

## 6. Decision Framework

| 狀態 | 意義 |
|---|---|
| 觀察中 | 尚未進入策略區 |
| 允許建立小部位 | 健康 / 股價 / 股性 / 有量 / 回撤 / 風險條件通過 |
| 等待確認 | 價格到位但資料或風險未確認 |
| 禁止加碼 | 品質失敗、波動過大、超過部位上限、資料異常 |
| 退出觀察 | 到達停損 / 停利 / Exit Rule |

---

## 7. Required Rules Before Buy

正式買入前必須定義：

1. 進場規則
2. 停止加碼規則
3. 退出規則
4. 最大部位
5. 單筆上限
6. 月度上限
7. 最大回撤容忍
8. 是否允許長持
9. 價格偏離 / 流動性檢查
10. Kill Switch
11. 健康 / 股價 / 股性 / 成交量篩選

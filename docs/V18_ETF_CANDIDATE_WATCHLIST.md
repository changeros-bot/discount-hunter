## 1. Rule

ETF 候選觀察名單不是正式買入名單。

正式 App Universe 暫時不擴大。候選 ETF 只用來：

1. 建立未來觀察池。
2. 支援 V18.1 回測。
3. 決定哪些 ETF 適合 DCA、哪些適合折扣加碼、哪些只適合現金管理。
4. 避免把個股風險錯誤用 ETF 解決，也避免 ETF 名單過度膨脹。

---

## 2. ETF Lifecycle

所有 ETF 必須經過：

```text
候選觀察
→ 回測
→ 策略定位
→ 顯示用 / DCA / 折扣加碼 / 現金管理
→ 正式納入
```

禁止：

```text
候選觀察 → 直接買入
```

## 4. App Classification

ETF 在 App 內不應全部顯示成同一種。

建議狀態：

| App Status | 意義 |
|---|---|
| 正式買入 | 已通過 Playbook + 回測 + 資金配置 |
| 候選觀察 | 有價值，但未通過回測或尚未定義規則 |
| 顯示用 | 只顯示價格 / 高點 / 回撤，不作買入提示 |
| 現金管理 | 用於停泊資金，不屬於折價獵人主買入 |
| 停用 | 暫不追蹤 |

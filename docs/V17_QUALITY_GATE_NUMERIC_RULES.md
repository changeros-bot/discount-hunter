# DCA 折價獵人 V17 Quality Gate｜數字門檻與稽核治理

目的：Quality Gate 不是資訊展示，而是自動化交易前的安全閘門。它決定每檔標的能否固定 DCA、逢低買進、進半自動草稿，以及未來是否能進有限自動交易白名單。

## 1. 股票 / xStocks 客觀財務門檻

評分：通過 = 2 分，觀察 = 1 分，失敗 = 0 分。客觀財務滿分 10 分。

| 項目 | 通過 | 觀察 | 失敗 |
|---|---|---|---|
| 營收成長 | 最近 TTM 或年度 YoY > 10%，且未連續兩期明顯放緩 | 0% 至 10%，或仍成長但增速明顯放緩 | < 0%，或連續兩期衰退 |
| 自由現金流 FCF | FCF 為正，且最近年度/TTM 不低於前一年 80% | FCF 為正但下滑超過 20%，或剛轉正未穩定 | FCF 為負，或由正轉負 |
| 毛利率 | 最近年度/TTM 毛利率不低於三年平均 -3 個百分點 | 低於三年平均 3 至 8 個百分點，或明顯承壓 | 低於三年平均超過 8 個百分點，或結構性惡化 |
| 資產負債表 | 現金/短投充足，淨負債可控，利息支出未明顯侵蝕營運 | 槓桿上升、現金下降或利息壓力升高，但未危及營運 | 流動性壓力、債務再融資風險、利息負擔明顯惡化 |
| CapEx / 營收紀律 | CapEx/營收比穩定，且 CapEx 成長未長期超過營收成長 2 倍 | CapEx 快速上升，需要觀察回報 | CapEx 連續高燒且營收/FCF 未跟上，疑似過度燒錢 |

## 2. 質化條件：必須有理由

評分：通過 = 2 分，觀察 = 1 分，失敗 = 0 分。質化滿分 8 分。

每一項不能只寫分數，必須寫一句理由，格式如下：

> 護城河：通過 — CUDA 生態系與開發者黏性仍在，短期未見有效替代。

| 項目 | 判定方式 |
|---|---|
| 產業領導地位 | 是否仍在核心產業位置，市占/供應鏈/客戶地位是否被破壞 |
| 護城河 | 技術、品牌、規模、資料、生態系、客戶轉換成本是否仍有效 |
| 管理層品質 | 執行力、資本配置、財務紀律、重大爭議 |
| 投資假設是否成立 | 當初納入折扣獵人的核心理由是否仍成立 |

## 3. 重大失敗項

以下三項任一項失敗，不得用總分硬蓋過去：

| 重大項目 | 失敗後處理 |
|---|---|
| 自由現金流 FCF | 最高只能是「觀察」；若同時營收衰退，降為「失敗」 |
| 資產負債表 | 最高只能是「觀察」；若有流動性危機，直接「失敗」 |
| 投資假設是否成立 | 若核心投資假設破壞，直接「失敗」 |

## 4. 總分判定

股票 / xStocks 滿分 18 分。

| Quality | 條件 | 權限 |
|---|---|---|
| 通過 | 14–18 分，且無重大失敗項 | 可 DCA、可逢低、可半自動草稿、可列自動交易白名單候選 |
| 觀察 | 9–13 分，或重大項目需觀察 | 可持有，DCA 可小額，逢低低優先或需人工確認，不進完全自動 |
| 未檢查 | 資料不足或 IPO/上市歷史不足 | 不進自動交易，只能人工確認 |
| 失敗 | 8 分以下，或核心假設破壞 | 停止新增資金，不產生草稿，不進自動交易 |

## 5. 既有部位處置

| Quality 變化 | 既有部位處置 | 新增資金 |
|---|---|---|
| 通過 → 觀察 | 不主動減碼；保留持倉 | 停止高優先逢低；只允許低優先或人工確認 |
| 觀察 → 失敗 | 進入退出/減碼檢查，不自動賣出 | 停止 DCA、停止逢低、停止草稿 |
| 未檢查 | 保留既有小額觀察 | 不進自動交易；逢低需人工確認 |
| 通過維持通過 | 持續原策略 | 可 DCA、可逢低、可草稿 |

## 6. 重新稽核觸發

| 觸發 | 動作 |
|---|---|
| 每季財報公布後 3 個工作天內 | 重新跑客觀財務區塊 |
| 每季治理審查 | 重跑完整 Quality Gate |
| 年度名單增減 | 重跑完整 Quality Gate + Asset Admission |
| 股價單月跌幅 > 30% 且非大盤系統性因素 | 觸發快速稽核 |
| CEO/CFO 變動、重大訴訟、監管事件、財務重編 | 觸發質化與重大失敗項稽核 |
| 信用/債務/流動性風險升高 | 觸發資產負債表與 FCF 稽核 |

## 7. BTC 簡化 Quality Gate

BTC 不使用股票財務表，改用 5 項，每項 0–2 分，滿分 10 分。

| 項目 | 通過 | 觀察 | 失敗 |
|---|---|---|---|
| 網路安全性 | 算力/安全性維持高水位，無重大協議風險 | 安全性或礦工經濟出現壓力 | 重大協議/安全事件 |
| 流動性 | 全球主要交易所流動性正常 | 流動性惡化或交易成本上升 | 大型交易所流動性斷裂 |
| 採用趨勢 | ETF/機構/支付/儲備敘事仍成立 | 採用停滯 | 採用敘事明顯反轉 |
| 監管風險 | 主要市場無全面封殺 | 局部限制升高 | 主要市場重大禁令 |
| 託管/交易所風險 | 自有風控與交易所風險可控 | 交易所/託管風險升高 | 資產安全出現重大風險 |

判定：8–10 通過；5–7 觀察；0–4 失敗。

## 8. ETF / QQQon 簡化 Quality Gate

ETF 不用單一公司財報，改用 5 項，每項 0–2 分，滿分 10 分。

| 項目 | 通過 | 觀察 | 失敗 |
|---|---|---|---|
| 指數代表性 | 仍代表 Nasdaq 100 / 核心科技成長組合 | 成分風格偏移需觀察 | 指數代表性不符策略 |
| 成分集中風險 | 集中但仍可接受 | 前幾大成分過度集中 | 單一/少數成分主導到失衡 |
| 長期成長邏輯 | 科技/平台/AI 長期邏輯仍成立 | 成長邏輯放緩 | 長期邏輯破壞 |
| 成本與流動性 | 費用率/流動性合理 | 流動性或成本略劣化 | 流動性不足或成本不合理 |
| 與核心組合角色 | 仍適合穩定 DCA | 需降低權重 | 不適合作為核心 DCA |

判定：8–10 通過；5–7 觀察；0–4 失敗。

## 9. SPCXon / 新上市標的規則

SPCXon 不是「非上市公司」處理，而是「新上市 / 交易歷史不足」處理。

| 項目 | 規則 |
|---|---|
| 52 週資料 | 未滿 52 週時不得用完整 52 週高點框架 |
| 高點基準 | 使用上市以來高點 |
| Quality 狀態 | 預設未檢查或觀察，不得直接通過 |
| DCA | 可保留 5U 小額 DCA |
| 逢低 | 必須人工確認資料源與上市以來高點 |
| 自動交易 | 不進自動交易白名單，直到至少兩季財報與足夠交易歷史 |

## 10. Asset Admission Process 銜接

Quality Gate 是每次階段升級的必要條件，不是只在最初納入時做一次。

| 階段 | Quality 要求 | 權限 |
|---|---|---|
| Research | 可未檢查 | 只研究，不交易 |
| Watchlist | 至少完成初審 | 觀察，不買或只提醒 |
| Paper Tracking | 至少觀察 | 紙上追蹤 |
| Small Position | 觀察以上 | 小額人工買入 |
| Core Candidate | 通過，且無重大失敗項 | 可 DCA / 可逢低 / 可草稿 |
| Core Asset | 連續兩次季度通過 | 可列自動交易白名單候選 |

## 11. 第一批稽核前置條件

在正式跑 BTC、QQQon、NVDAon、TSMon、AVGOon、GOOGLon 前，必須先完成：

1. 確認資料來源。
2. 使用上述客觀門檻。
3. 每個質化分數寫一句理由。
4. 檢查重大失敗項。
5. 寫出既有部位處置。
6. 輸出交易權限。
